# booked.rentals — Setup & Architecture Guide

Last updated: April 1, 2026

---

## Project Overview

booked.rentals is a static content site for short-term rental hosts, built with **Astro v6** and deployed on **Netlify**. Content is managed via **Sveltia CMS** (Git-based, no database). All content lives as markdown files in the repo.

## Content Collections

Defined in `src/content.config.ts` using Astro v6 `glob` loaders with Zod validation.

### Articles (`src/content/articles/`)
- Fields: title, date, category, excerpt, image (optional), featured, draft
- Categories must match a Topic title exactly (for article counts)
- File naming: `YYYY-MM-DD-slug.md`

### Products (`src/content/products/`)
- Fields: title, review, rating, category, image (optional), affiliate_url (optional), featured
- File naming: `slug.md`

### News (`src/content/news/`)
- Fields: title, date, excerpt, draft
- File naming: `YYYY-MM-DD-slug.md`

### Topics (`src/content/topics/`)
- Fields: title, description, icon (SVG string), visible, sort_order
- Controls the "Explore Topics" section on the homepage
- Article counts are computed dynamically by matching `topic.title` to `article.category`
- Visibility and ordering are CMS-controllable
- File naming: `slug.md`

### Site Settings (`public/content/settings.yml`)
- Fields: title, tagline, hero_headline, hero_headline_emphasis, hero_subtitle, hero_cta_button, hero_disclaimer, cta_headline, cta_button_text, subscriber_count, pinterest_url, instagram_url
- Single YAML file (not a collection), read at build time by `index.astro`
- Editable via CMS under "Site Settings > General"

## Typography & Fonts

### Font Stack

| Role | Font | Source | Fallback |
|------|------|--------|----------|
| Body text | Merriweather | Self-hosted WOFF2 (`public/fonts/`) | Georgia, serif |
| Headings, UI, nav, buttons | TeX Gyre Heros | Self-hosted WOFF2 (`public/fonts/`) | Helvetica, Arial, sans-serif |
| Logo wordmark | Helvetica (as SVG paths) | Inline SVG (no font dependency) | N/A |
| Tagline | Helvetica (as SVG paths) | Inline SVG (no font dependency) | N/A |

### Why These Choices

- **TeX Gyre Heros** is a metrically-identical open-source clone of Helvetica (GUST e-foundry, LaTeX Project Public License). Self-hosting it ensures consistent rendering on all platforms.
- **Logo and tagline** are rendered as SVG with text converted to vector paths (outlines). This guarantees pixel-perfect Helvetica rendering everywhere with zero font dependency.
- **No Google Fonts dependency.** All fonts are self-hosted from `public/fonts/` for speed and privacy.

### Adding Font Files

Drop WOFF2 files into `public/fonts/`:
- `merriweather-latin.woff2` (already present)
- `tex-gyre-heros-regular.woff2` (to be added)
- `tex-gyre-heros-bold.woff2` (to be added)

`@font-face` declarations are in `global.css` at the top of the file.

## Homepage Topics Grid Logic

The "Explore Topics" section uses a smart row-balancing algorithm:

- **1–4 topics**: Single row
- **Even count (6, 8, 10...)**: Two equal rows (e.g., 6 → 3+3)
- **Odd count (5, 7, 9, 11...)**: Round up to nearest even (x), top row = (x/2)−1, bottom row = (x/2)
  - 5 → 2+3
  - 7 → 3+4
  - 9 → 4+5
  - 11 → 5+6

CSS uses a `--cols` custom property set inline by the Astro template, so the grid adapts automatically. On tablet (769–1024px) it falls back to 2 columns; on mobile (<768px) it stacks to 1 column.

## Sveltia CMS

### Authentication

Sveltia CMS uses **GitHub OAuth** (not Netlify Identity). Configure in:
- Netlify Dashboard → Site settings → Access & security → OAuth

### How to Access

1. Go to `https://homepage-rebuild--booked-rentals.netlify.app/admin/`
2. Authenticate via GitHub OAuth
3. Edit content collections or site settings

### CMS Configuration

All config lives in `public/admin/config.yml`. Key settings:
- `backend.branch`: `homepage-rebuild` (change to `main` when merging)
- `site_url`: Branch preview URL (update to production URL when merging)
- `media_folder`: `public/images/uploads`

- **Logo box**: `linear-gradient(135deg, var(--purple) 0%, var(--teal) 100%)`
- **Gradient text** (hero emphasis, `.gradient-text`): purple→teal with `-webkit-background-clip: text`
- **CTA buttons**: `linear-gradient(135deg, var(--purple) 0%, var(--pink) 100%)`

### Responsive Breakpoints

- **480px**: Small phones
- **768px**: Tablets (hamburger menu triggers)
- **1024px**: Desktop (topics grid uses `--cols`)

## CMS-Editable vs. Code-Controlled

| What | Editable in CMS? | How to change |
|------|:-:|---|
| Article/product/news content | Yes | CMS or edit markdown directly |
| Topic visibility, order, description, icon | Yes | CMS under Topics collection |
| Hero headline, subtitle, CTA text | Yes | CMS under Site Settings > General |
| Email capture headline, button text | Yes | CMS under Site Settings > General |
| Subscriber count display | Yes | CMS under Site Settings > General |
| Navigation menu items | No | Edit `Navbar.astro` |
| Footer navigation | No | Edit `Footer.astro` |
| Page section order/layout | No | Edit page `.astro` files |
| Design tokens (colors, fonts) | No | Edit `global.css` |

## Building & Deploying

### Local Development

```bash
cd staging/
npm run dev          # Start dev server at localhost:4321
npm run build        # Build static site to dist/
npm run preview      # Preview built site locally
```

**Note:** If running build on the mounted filesystem causes EPERM errors, copy to `/tmp` first:
```bash
cp -r staging/ /tmp/booked-build && cd /tmp/booked-build && npm install && npm run build
```

### Netlify

Builds automatically on push to any branch. Config in `netlify.toml`:
- Build command: `npm run build`
- Publish directory: `dist`
- Node version: 22
